## Data Storage

**Databases:**
- PostgreSQL - Primary persistent data store
  - Connection: Via `DATABASE_URL` environment variable
  - Recommended Provider: Neon Postgres (in-memory fallback for local dev)
  - Client: Prisma 6.19.2 (ORM)
  - Schema location: `prisma/schema.prisma`
  - Models: `User`, `Team`, `TeamMember`, `Match`, `AuthRateLimit`

**In-Memory Fallback:**
- RAM-based store when `DATABASE_URL` is absent
  - Location: `src/lib/data.ts`
  - Usage: Local development, unit/component tests
  - Shared state across requests (not production-safe)

**Client-Side Storage:**
- localStorage - Theme preference only
  - Key: `office-8-ball-theme` (configurable in `src/app/layout.tsx`)
  - Values: "light" | "dark" | "sepia"
  - Provider: Browser built-in (no external service)
  - Location: `src/components/theme/theme-provider.tsx`

**File Storage:**
- No external file storage (S3, Cloud Storage, etc.)
- User avatars stored as URL string in database (`avatarUrl` field)
- No multipart form uploads

**Caching:**
- None configured (no Redis, Memcached, or HTTP caching headers)
- Prisma client uses in-process caching only

## Authentication & Identity

**Auth Provider:**
- Custom credentials-only implementation via Auth.js (next-auth 4.24.13)
  - Location: `src/lib/auth.ts`
  - Provider: CredentialsProvider (username/password)
  - Session strategy: JWT (signed with `NEXTAUTH_SECRET`)
  - No OAuth, social login, or third-party identity provider

**Authentication Flow:**
1. POST `/api/auth/register` - Signup with username/password (Zod validated)
2. POST `/api/auth/callback/credentials` - Login via next-auth (generated by framework)
3. GET `/api/auth/session` - Retrieve current session (automatic by next-auth)
4. Middleware (`middleware.ts`) - Protect routes by checking session

**Password Security:**
- Hashed with bcryptjs 3.0.3 (bcrypt algorithm)
- Location: `src/lib/auth-validation.ts` - Hash during signup
- Verified during login in `src/lib/auth.ts` via `compare()` function

**Rate Limiting:**
- Custom implementation in `src/lib/auth-rate-limit.ts`
- Stored in `AuthRateLimit` table in database
- Blocks repeated login failures per username/IP
- Configurable block levels and time windows

## Environment Configuration

**Required env vars for production:**
- `DATABASE_URL` - PostgreSQL connection (Neon format)
- `NEXTAUTH_SECRET` - Auth session signing secret (strong random value)
- `NEXT_PUBLIC_APP_ENV` - "production" (sent to client)

**Optional env vars:**
- `NEXTAUTH_URL` - Session callback URL (auto-detected by Vercel)
- `PLAYWRIGHT_BASE_URL` - E2E test server (local dev only)
- `CI` - Set by GitHub Actions

**Secrets location:**
- Vercel: Environment variables in project dashboard (linked to GitHub repo)
- Local dev: `.env.local` (copied from `.env.example`, never committed)
- Production: `.env.production.example` shows required variables (no actual values committed)

**Configuration validation:**
- `src/lib/auth.ts` - Functions check `hasDatabaseUrl()`, `hasAuthSecret()`, `isAuthAvailable()`
- Routes return 503 (service unavailable) if auth requirements not met

## API Routes

**Internal APIs** (all require authentication via `getAuthenticatedUser()`):
- `GET/POST /api/matches` - Fetch/register match winners
  - Location: `src/app/api/matches/route.ts`
  - Database: Reads/writes to `Match` table
- `GET /api/scoreboard` - Aggregated stats (derived from matches)
  - Location: `src/app/api/scoreboard/route.ts`
  - No direct API calls, data calculated in-process
- `POST /api/auth/register` - User signup
  - Location: `src/app/api/auth/register/route.ts`
- `GET/POST /api/teams` - List/create teams
  - Location: `src/app/api/teams/route.ts`
- `GET/POST /api/teams/[id]` - Get/update team by id
  - Location: `src/app/api/teams/[id]/route.ts`
- `POST /api/teams/[id]/archive` - Archive team
  - Location: `src/app/api/teams/[id]/archive/route.ts`
- `GET /api/users` - List users (protected)
  - Location: `src/app/api/users/route.ts`
- `GET /api/profile` - Current user profile
  - Location: `src/app/api/profile/route.ts`

**Next.js Auth.js Route:**
- `GET/POST /api/auth/[...nextauth]` - Auth.js framework routes (login, logout, etc.)
  - Location: `src/app/api/auth/[...nextauth]/route.ts`
