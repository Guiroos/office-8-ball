## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Display | clamp(4rem, 12vw, 6rem) | 700 | 1.2 | Hero headings, 1st place podium card title |
| Heading | clamp(1.5rem, 4vw, 2.5rem) | 700 | 1.2 | Section headers (Ranking, Team Detail, Membros, Últimas Partidas, Confrontos Diretos) |
| Body | 1rem (16px) | 400 | 1.5 | Default text, team names in standings, match details, stat values |
| Label | 0.875rem (14px) | 400 | 1.0 | Field labels, badges, metadata, captions, stat tile labels, rank indicators |

**Notes:**
- Display and Heading use weight 700 (bold); Body and Label use weight 400 (regular).
- Font sizes use clamp() for fluid scaling (Display, Heading) or fixed values (Body, Label).
- Heading role merges previous Subtitle, Title, and Headline variants — all section-level headings use this single role.
- Label role merges previous Label, Body Small, and Caption variants — all small/metadata text uses this single role.
- All heading roles apply letter-spacing adjustments automatically via @utility rules in globals.css.

**Source:** `src/app/globals.css` @utility declarations and `src/app/tokens.css` font-size vars.

## Component Patterns

### Ranking Page Layout

**Podium Section (Top 3):**
- Grid layout: `grid-cols-3` with center-aligned podium styling
- 1st place (center): Larger badge, gold highlight, trophy icon
- 2nd place (left): Smaller badge, silver/neutral highlight, medal icon
- 3rd place (right): Smaller badge, bronze/muted highlight, medal icon
- Each podium card displays: team name + wins + win rate % + current streak
- Use `StatTile` for stat display within podium cards

**Standings List (Rank 4+):**
- Compact table or list layout below podium
- Rows show: rank # + team name + W/L + win rate % + streak + total matches
- Each row clickable → navigate to `/times/[teamId]`
- Use consistent spacing (md/lg) between rows

**Filter Tabs:**
- Two tabs: "Solo" | "Duplas" (filter by `team.type`)
- Tab styling: button-like appearance, active tab uses primary color, inactive uses muted foreground
- Tabs live above podium section

**Layout Structure:**
```
┌─ Filter Tabs (Solo | Duplas)
├─ Podium Section (1st/2nd/3rd)
└─ Standings List (#4+)
```

### Team Detail Page Layout

**Header Section:**
- Team avatar (initials or colored badge) on left
- Team name (large title or colored badge) on left
- "Invite Member" CTA button on right
- Background: optional surface-emphasis or gradient

**Stats Grid:**
- Four StatTiles in 2x2 grid (or responsive): Total Wins, Total Losses, Win Rate %, Current Streak
- Use `StatTile` component from `src/components/primitives/stat-tile.tsx`
- Tile tone: "default" (light backgrounds, dark text)

**Roster Section:**
- Section header: "Membros" (via `SectionHeader` component)
- List of members with: avatar + name + role (Owner/Member)
- Simple list with vertical spacing (gap-md)

**Recent Matches Section:**
- Section header: "Últimas Partidas"
- Display last 3 matches in compact format: date + opponent + result (W/L)
- "Ver histórico completo" link at bottom (navigates to historical view, Phase 5 scope)

**Head-to-Head Section:**
- Section header: "Confrontos Diretos"
- Dropdown to select opponent team (pre-loaded with primary rival — team with most direct matches)
- Display H2H stats: W/L in head-to-head + win rate vs opponent + date of last match
- Use two-column layout: "Seu Time" (left) vs "Adversário" (right)

**Layout Structure:**
```
┌─ Header (Avatar + Name + CTA)
├─ Stats Grid (4 tiles)
├─ Roster Section (members list)
├─ Recent Matches Section (last 3 + link)
└─ H2H Comparison Section (dropdown + stats)
```

### Reusable Components

**StatTile** (`src/components/primitives/stat-tile.tsx`):
- Props: `label` (caption), `value` (string/number/ReactNode), `description` (optional), `tone` ("default" | "inverse")
- Used in: ranking podium, team detail stats grid
- Styling: rounded-lg border with surface-emphasis background (default tone)

**SectionHeader** (`src/components/primitives/section-header.tsx`):
- Props: `eyebrow`, `title`, `description`, `className`, `titleClassName`, `descriptionClassName`
- Used in: ranking page title, team detail roster/matches/H2H section headers

**Card & CardContent** (`src/components/ui/card.tsx`):
- Wrapper for grouped content areas
- Used for: podium cards, team detail main container

**Badge** (`src/components/ui/badge.tsx`):
- Display streak indicators, rank indicators, team type badges
- Variants available via CVA

**Button** (`src/components/ui/button.tsx`):
- Primary: green (primary color), used for "Create Team", "Invite Member", "Save" CTAs
- Secondary: muted surface, used for tertiary actions
- Destructive: red (danger color), used for delete/remove only

## Layout & Responsive Behavior

### Ranking Page

**Desktop (lg+):**
- Podium grid: `grid-cols-3` with equal columns
- Standings list: full-width table or card list
- Tabs above all content

**Tablet (md):**
- Podium: `grid-cols-3` still works (responsive card sizes)
- Standings: compact list view

**Mobile (sm):**
- Podium: `grid-cols-1` (stack vertically) or carousel
- Standings: single-column list, compact rows
- Tabs remain two-button row

### Team Detail Page

**Desktop (lg+):**
- Header in single row: avatar + title + CTA
- Stats grid: 2x2 layout
- Roster, matches, H2H in vertical stack

**Tablet (md):**
- Header single row still works
- Stats grid: responsive (2x2 or 1x4)
- Stack other sections vertically

**Mobile (sm):**
- Header: avatar/title on left, CTA stacked below
- Stats grid: 1x4 vertical stack
- All sections single-column

## Interactions & States

### Ranking Page

**Hover States:**
- Ranking row hover: `shadow-md` (from `shadow-sm`), slight background lift
- Podium card hover: same shadow escalation
- Team name link hover: underline or color change to primary

**Active States:**
- Selected filter tab: background primary color, white/inverse text
- Unselected tab: muted background, muted foreground

**Loading States:**
- Skeleton cards matching podium/list structure
- Pulse animation (via Tailwind's default or custom `tw-animate-css`)

**Error States:**
- Toast notification (via `sonner`): "Erro ao carregar ranking. Tente novamente."
- Page fallback: "Não foi possível carregar o ranking. Atualize a página."

### Team Detail Page

**Hover States:**
- Member row hover: subtle background highlight
- Match row hover: shadow escalation
- Dropdown hover: standard Radix UI hover styling

**Loading States:**
- Skeleton for header, stats, roster, matches, H2H
- Each section can skeleton independently (RSC streaming)

**Error States:**
- If team not found (404): "Time não encontrado" message with back link
- If stats fail to load: "Não foi possível carregar as estatísticas do time" with retry

**Member Management** (future, Phase 1 already implemented member add/remove UI):
- Remove member: confirmation dialog, "Tem certeza? {Member Name} será removido do time."
- After removal: toast success "Membro removido" + update roster list

## Shadow System

Shadows composed via class pairs: `shadow-{size} shadow-{color}/{opacity}`

| State | Size | Color Example | Classes |
|-------|------|---------------|---------|
| Rest | sm | gold/35 | `shadow-sm shadow-gold/35` |
| Hover | md | gold/40 | `shadow-md shadow-gold/40` |
| Active/Press | xs | gold/25 | `shadow-xs shadow-gold/25` |

**Usage:**
- Podium cards: `shadow-sm shadow-gold/35` at rest, `shadow-md` on hover
- Regular ranking rows: `shadow-xs shadow-foreground/10` at rest, `shadow-sm` on hover
- Stat tiles: `shadow-sm shadow-foreground/8` (subtle)
